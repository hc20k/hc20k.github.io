CELLULAR AUTOMATA

What exactly are cellular automata?


Cellular automata is actually the plural version of a cellular automaton (aka CA for short). A cellular automaton is a model of a system of "cell" objects with the following characteristics:

  • Each cell lives on the same grid
  • Each cell has its own state (on or off, 0 or 1)
  • Each cell has a neighborhood, which are the cells that are close to it.

    Generations

    In order for cellular automata to be cellular automata, they need to have generations. The CA have to live for a certain amount of time, and have to have rules indicating whether they can survive a generation or not.

    A cell's new state in future generations are determined by the cells neighbors in the current generation. We can calculate a new state by looking at its neighbors' states and using a ruleset.

    A ruleset is 8 bits because there are 8 possible neighborhood configurations. Here's an example ruleset (Rule 90):

    0, 1, 0, 1, 1, 0, 1, 0

    The Game of Life

    In 1970, mathematician John Conway invented his "Game of Life". The Game of Life uses two dimensional arrays instead of just one, and its goal is to simulate the natural world in code. It simulates different biological systems like reproduction, life, and death. Scientist Martin Gardner explained Conway's goals with his Game of Life pretty well:
    1. There should be no initial pattern for which there is a simple proof that the population can grow without limit.
    2. There should be initial patterns that apparently do grow without limit.
    3. There should be simple initial patterns that grow and change for a considerable period of time before coming to an end in three possible ways: fading away completely (from overcrowding or becoming too sparse), settling into a stable configuration that remains unchanged thereafter, or entering an oscillating phase in which they repeat an endless cycle of two or more periods.

    But how does it actually work?

    Since we use a 2D grid with the Game of Life, and all of the cells follow the rules of cellular automata, an example grid would look like this:



    In the Game of Life, a cell has a neighborhood of 9 cells, including itself. Since we have nine cells in a neighborhood, there's 9 bits, which means there's 512 possible neighborhoods. Because it would be inconventient to provide an outcome for each individual cell, the Game of Life uses rules for the neighborhoods.
    1. A cell will die if it has 4 or more alive neighbors, or if it has 1 or less neighbors.
    2. A cell will come back to life if it has exactly 3 live neighbors.
    3. If none of these conditions are met, the cell will maintain its state.
